## Исправления
- Остановка namespace теперь корректно удаляет все его контейнеры — медленно останавливающееся приложение (например, eapps) больше не остаётся в состоянии «Exited» после выключения, а сеть namespace надёжно удаляется.
- Удаление рабочего пространства теперь вычищает всё внутри него — Docker-тома данных каждого namespace (PostgreSQL, MongoDB, …) и записи в базе, а не только файлы на диске.
- Встроенный сервер конфигурации (десктоп) теперь работает только пока namespace активен и освобождает порт при остановке, поэтому перезапуск приложения при остановленном namespace больше не падает с «address already in use».
