## New features
- **Platform variables for additional apps.** The environment (and command / init) values of an `additionalApps` entry can now reference platform-managed values via new `${VAR}` placeholders, so a config-driven service integrates with ECOS authentication and messaging without hardcoding secrets in the workspace config: `${JWT_SECRET}`, `${OIDC_SECRET}`, `${WEB_URL}` (the namespace's public base URL), `${RMQ_USER}` / `${RMQ_PASSWORD}`, `${KK_HOST}` and `${ADMIN_PASSWORD}`, alongside the existing infrastructure host/port variables. For example, `AUTH_JWTSECRET: "${JWT_SECRET}"` makes a service validate the JWT forwarded by the gateway — using the same secret as the built-in webapps.
