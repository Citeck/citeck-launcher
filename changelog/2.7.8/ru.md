## Исправления
- Исправлено появление экрана «Docker недоступен» на macOS даже при запущенном Docker Desktop. Теперь лаунчер определяет Docker так же, как это делает CLI `docker` — через активный docker-контекст (Docker Desktop, colima, Rancher Desktop) — и при необходимости перебирает дополнительные типовые пути сокета, а не проверяет только путь по умолчанию.
