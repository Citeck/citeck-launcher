## Новые возможности
- **Пользовательские ссылки в боковой панели.** Объявляйте свои быстрые ссылки в конфигурации воркспейса (`workspace-v1.yml` → `links:`), каждую с необязательным списком зависимостей `dependsOn`. Ссылка скрывается, пока зависимость отсутствует в неймспейсе, отключается, пока зависимость не запущена, и активна в остальных случаях. Пользовательские ссылки отображаются внизу боковой панели.
- **Редактирование конфигурации приложения из командной строки.** `citeck edit <app>` открывает эффективную конфигурацию приложения в вашем `$EDITOR` и сохраняет её как пер-аппный оверрайд (как `kubectl edit`). `--file <путь>` вместо этого правит смонтированный конфиг-файл, например `application-launcher.yml`, `--list-files` выводит список правимых файлов, `--reset` восстанавливает сгенерированное значение по умолчанию, а `--from <файл|->` задаёт содержимое без запуска редактора.

## Изменения
- **Изменения смонтированных конфиг-файлов теперь доходят до работающего приложения.** Сохранение изменения в смонтированном конфигурационном файле (например, `application-launcher.yml`) теперь пересоздаёт соответствующий контейнер, чтобы новое содержимое применилось; ранее правка сохранялась, но не вступала в силу до следующего постороннего изменения. При обновлении контейнеры веб-приложений один раз пересоздаются (меняется их хеш развёртывания) — действий не требуется.
- Лимит памяти Keycloak повышен с 1 ГБ до 1,5 ГБ.
- Лимиты памяти теперь принимают дробные единицы (например, `1.5g`).
