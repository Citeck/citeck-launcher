## 新功能
- 当 Docker 数据目录磁盘空间不足时，启动器会在容器开始故障之前发出警告。

## 修复
- “强制更新并启动”现在会立即响应，并通过 Git 强制检查新的捆绑包版本，同时复用已下载的发布镜像，而不是重新拉取。
- 停止时卡住的应用现在会自行恢复，不再冻结命名空间。
- 如果 Git 操作停滞，命名空间重新加载不再无限期挂起。
- 容器的自动重启会复用本地镜像，因此绝不会悄悄更改版本，也不会在镜像仓库不可用时失败。
- 已删除命名空间遗留的 Docker 容器、卷和网络现在会在启动时自动清理。
- 活动命名空间始终显示其自身的容器，不再出现命名空间不匹配。
- 当您将应用最小化到托盘或返回欢迎界面时，附加窗口（日志、编辑器）现在会关闭。

## 变更
- 命名空间通过表单进行编辑；已移除原始 YAML 编辑器和齿轮图标的右键菜单。
