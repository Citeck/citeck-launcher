## 新機能
- Docker のデータディレクトリの空き容量が少なくなると、コンテナーが失敗し始める前にランチャーが警告します。

## 修正
- 「強制更新して起動」が即座に応答するようになり、Git 経由で新しいバンドルバージョンを強制的に確認しつつ、すでにダウンロード済みのリリースイメージを再取得せずに再利用します。
- 停止時にスタックしたアプリが自動的に復旧し、ネームスペースをフリーズさせなくなりました。
- Git 操作が停滞しても、ネームスペースの再読み込みが無期限にハングしなくなりました。
- コンテナーの自動再起動はローカルイメージを再利用するため、バージョンが知らないうちに変わったり、レジストリ障害時に失敗したりすることはありません。
- 削除されたネームスペースに残っていた Docker のコンテナー・ボリューム・ネットワークが、起動時に自動的にクリーンアップされるようになりました。
- アクティブなネームスペースは常に自身のコンテナーを表示し、ネームスペースの不一致がなくなりました。
- アプリをトレイに最小化したりウェルカム画面に戻ったりすると、追加のウィンドウ（ログ、エディター）が閉じるようになりました。

## 変更
- ネームスペースはフォームから編集します。生の YAML エディターと歯車アイコンの右クリックメニューは削除されました。
