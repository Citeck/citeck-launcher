## Новые возможности
- Масштабирование интерфейса штатными средствами веб-вью.
- Перезапуск отдельного приложения из контекстного меню — доступен в любом состоянии приложения.

## Исправления
- Серверная установка с демоданными теперь действительно импортирует демоснапшот — раньше namespace создавался, но запускался с пустыми данными.
- Серверная установка теперь сохраняет введённый при настройке логин к приватному реестру, а не молча отбрасывает его.
- Больше совместимости с лаунчером 1.x для мигрированных или вручную отредактированных конфигов: пользователи namespace по умолчанию (admin + fet), настройки bundle/workspace и значения по умолчанию для probe/log-startup.
- Перед добавлением первого секрета теперь предлагается задать мастер-пароль, а не возникает ошибка.
- Quick Start закрепляет новейшую версию за конкретным релизом, обновляет список доступных версий после разблокировки секретов и открывает панель namespace сразу после запуска.
- Пользовательские репозитории workspace восстанавливаются автоматически после добавления или исправления токена доступа — перезапуск не нужен.
- Сохранённые учётные данные реестра сразу привязываются к своему хосту реестра.
- Значения секретов теперь чётко помечаются как Token или Password.
- Диалог снапшотов: общий заголовок и автоматическое обновление по завершении экспорта или импорта.
- Более контрастная панель деталей приложения; редактор конфигурации теперь следует теме приложения.
- Более читаемый текст логов и подсказка, почему «Удалить всё» недоступно.
- Удаление workspace теперь также удаляет привязки его учётных данных реестра.
